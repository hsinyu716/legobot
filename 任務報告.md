# TTRA 學科題庫線上測驗 — 任務報告

## 一、任務目標

依《學科題庫公告.pdf》設計一個出題網頁：

1. 將公告中的 **300 題** 建立成題庫。
2. 以 [waterball_453 / cursor_ttra.md](https://github.com/hsinyu716/waterball_453/blob/main/cursor_ttra.md) 的答案為參考，**複核並修正錯誤**。
3. 將「有變動的題目＋答案」輸出成 md 檔。
4. 可**從 300 題隨機抽 20 題**做測驗。
5. 成品可**直接放到 GitHub Pages 網頁執行**。

---

## 二、處理流程

### 1. 解析題庫 PDF
- PDF 共 36 頁、300 題，為 Excel 匯出的表格，**程式積木 / 齒輪 / 槓桿等題目以圖片呈現**，純文字無法完整表達。
- 用 `pdftotext -bbox` 取得每個文字的版面座標，定位全部 300 個題號（左欄純數字，逐頁累加比對 1→300，零遺漏）。
- 用 `pdftoppm` 將 36 頁渲染成 200 DPI 影像，並偵測表格的**水平格線**作為每一題的上下邊界。
- 依格線精準切出每一題的完整影像（題目＋四個選項＋所有圖示），共 **300 張** `img/q1.png ~ q300.png`。逐頁格線數與題數完全吻合。

### 2. 解析參考答案
- 從 cursor_ttra.md 解析出全部 **300 題**的參考答案（選項編號 1–4 ＋選項文字），無缺漏、無衝突。
- 答案分布：選項 1×98、2×89、3×64、4×49（並非全部都是選項 1）。

### 3. 答案對位驗證
- 程式比對「參考答案的選項編號」與「PDF 中該編號的實際選項文字」。
- 47 處差異**全為**標點全形/半形、異體字（値/値、長/⻑）或縮寫，**無一指向錯誤欄位** → 確認編號對位正確。

### 4. 逐題人工複核
- 將 300 題分 **15 批**，逐題依**題目影像**與領域知識（感應器特性、齒輪比、槓桿、滑輪、迴圈/條件/變數、馬達控制、重心等）判讀答案正確性，僅在高度確信有誤時才修正。
- 對所有被標記的題目再次親自開圖確認後才定案。

## 三、修正結果

複核 300 題後，發現並修正 **3 題**，其餘 297 題維持原答案。

| 題號 | 原答案 | 修正後 | 原因 |
|:---:|:---:|:---:|:---|
| 53 | 4 以上皆非 | **1 小齒輪帶動中齒輪** | 增加扭力須降速：小齒輪（主動）帶動較大齒輪（從動），選項 1 正確；原「以上皆非」錯誤。 |
| 155 | 3 角度感應器 | **4 濕度感應器** | 偵測牆壁時，濕度感應器完全無關、最不適用；原答案角度感應器較不精確。（此題屬偏弱題，已於 corrections.md 註明） |
| 176 | 2 太近會誤判 | **1 對著空氣照射會認為白色** | 題目問「何者**為非**」，選項 2/3/4 皆為正確敘述；對空氣無反射應讀為暗（黑）而非白，選項 1 才是錯誤敘述。 |

> 完整勘誤說明見 [`corrections.md`](corrections.md)。

## 四、成品（交付物）

```
ttra-quiz/
├── index.html        # 測驗網頁：隨機抽 20 題、即時評分、逐題解答（RWD）
├── questions.json    # 300 題題庫（題號、圖片、選項、答案；已含修正）
├── corrections.md    # 參考答案勘誤與修正紀錄（變動的題目＋答案）
├── 任務報告.md        # 本檔
├── README.md         # 部署與使用說明
└── img/              # q1.png ~ q300.png（每題完整影像）
```

### 網頁功能
- 🎲 每次從 300 題**隨機抽 20 題**，可隨時「重新抽題」。
- ✅ 交卷後立即計分，逐題標示正確/錯誤並顯示正解。
- 📱 支援手機/平板/桌機；純靜態，無需後端或建置。

## 五、部署到 GitHub Pages

1. 將 `ttra-quiz/` 內所有檔案（含 `img/`）上傳到 GitHub 儲存庫根目錄（或 `docs/`）。
   - 本資料夾已 `git init` 並完成首次 commit，只需 `git remote add origin <網址>` 後 `git push`。
2. **Settings → Pages → Source** 選 `Deploy from a branch`，Branch 選 `main`、資料夾 `/ (root)`（或 `/docs`）。
3. 約 1 分鐘後，以 `https://<帳號>.github.io/<儲存庫名>/` 開啟即可測驗。

### 本機預覽
```bash
cd ttra-quiz
python3 -m http.server 8000      # 開啟 http://localhost:8000
```
（直接以 `file://` 開啟會被瀏覽器擋下 `fetch`，請用本機伺服器或 GitHub Pages。）
